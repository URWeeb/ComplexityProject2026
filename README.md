# ComplexityProject2026

Алгоритмический проект по курсу Сложность Вычислений на тему "Выполнимость 3-КНФ"

## Описание структуры проекта:

- Теоретическая часть проекта будет изложена в директории TheoryData.

- Программная реализация будет разбита на несколько папок(src для *.cpp файлов, include для .hpp файлов), связанных с помощью CMake для сборки на различных ОС(целевой являются дистрибутивы Linux или Unix-подобные BSD-системы и MacOS). Для проверки корректности написанного будет использоваться Unit-тестирование(библиотека GoogleTest) и CI/CD.

## Гайд по запуску и использованию

Первоначально вам надо будет установить ряд зависимостей для банального запуска программы.

### Debian-Based(Debian, Ubuntu, Mint, e.t.c.)
```
sudo apt update
sudo apt install gcc cmake make 
```

### RedHat-Based(RHEL, Fedora, e.t.c.)
```
sudo dnf update
sudo dnf install gcc cmake make
```

### Arch-Based(Arch, CachyOS, Manjaro, SteamOS, e.t.c)
```
sudo pacman -Syu
sudo pacman -S gcc cmake make
```

### MacOS
GCC здесь - это просто alias для Clang, так что можно его не устанавливать, если пользовались Xcode раннее.
```
brew update
brew install gcc cmake make
```

После установки зависимостей достаточно просто скопировать этот репозиторий, а далее пользоваться следующими шорткатами
(для больших подробностей рекомендую почитать stages в Makefile).
### Сборка проекта
```
make build
```

### Запуск тестов
```
# Запуск без санитайзеров или с ними
make test
make test SANITIZE=address
```

### Запуск со стандартными санитайзерами
```
make sanitized
```

### Очистка от лишних файлов(директории сборки)
```
make clean
```

## Как пользоваться программой после сборки
```
cd build
# Не надо копировать, напишите ручками!!!
./levin_sat <(1)> <(2)> <(3)> <(4)> <(5)>
```

Здесь соответствующие пункты означают следующее:
1. Кол-во переменных (default = $20$)
2. Кол-во дизъюнктов в формуле (default = $85$)
3. Сид для рандомной генерации (default = $42$)
4. Максимальное кол-во раундов для универсального поиска Левина (default = $50000$)
5. Вывод логов во время работы кода (default = $true$)

Помимо этого, есть аргументы по умолчанию, поэтому работать будет и такой вариант:
```
cd build
./levin_sat
```

**Примечание**

Формулы нужно вводить в формате [DIMACS](https://jix.github.io/varisat/manual/0.2.0/formats/dimacs.html). Параметры p cnf и т.д. вводить не надо.
